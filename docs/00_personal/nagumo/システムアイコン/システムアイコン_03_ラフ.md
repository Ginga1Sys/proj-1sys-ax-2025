# システムアイコン ラフ案（上位3要素）

対象: `proj-1sys-ax-2025` システム用のアイコン検討ラフ。
頭文字は仮に「AX」をプレースホルダーとして使用（後で確定名に置換）。

配色（暫定）
- Primary: #0A84FF（アクセント）
- Neutral: #0B1220（濃い輪郭）
- Background: #FFFFFF（塗り/抜き兼用）
- Success: #22C55E（チェック/信頼）

## A. Hub & Nodes（統合・連携・自動化）
- 意図: 複数システム/データをハブで統合し、効率的に連携する性質を象徴。
- 要素: ノードとリンク、中心ハブ、わずかな進行矢印（スピード）。
- 使い所: プラットフォーム性、ワークフロー自動化、連携の強みを出したいとき。

## B. Monogram + Negative Space（頭文字の記号化）
- 意図: システムの頭文字（例: AX）を抽象化し、余白で矢印=進行/成果を示す。
- 要素: モノグラム、ネガティブスペースの矢印、角丸スクエア（アプリ感）。
- 使い所: ブランド想起を強めたいとき、単体でアプリアイコン化。
- SVG（最小ラフ）:
```svg
<svg width="256" height="256" viewBox="0 0 256 256" xmlns="http://www.w3.org/2000/svg">
  <rect width="256" height="256" rx="48" ry="48" fill="#0B1220"/>
  <!-- AX（仮） -->
  <text x="50%" y="52%" text-anchor="middle" dominant-baseline="middle"
        font-family="Segoe UI, Roboto, Arial" font-size="108" font-weight="700" fill="#FFFFFF">AX</text>
  <!-- 余白の矢印（切り抜き風）：白で重ねてネガティブスペース感を演出 -->
  <path d="M156 84 L196 128 L156 172" fill="none" stroke="#FFFFFF" stroke-width="18" stroke-linecap="round" stroke-linejoin="round"/>
</svg>
```

注: 実制作時は文字輪郭化（パス化）して字間/太さを微調整。頭文字確定後に置換可能。

- 要素: シールド、チェック、クラウド輪郭。

### 次のステップ
- 上記3案のどれを深掘りするか選定（1–2案）。
- 配色のブランド固有値（メイン/サブ/背景）を確定。
- マージン/余白/角丸半径/線幅のガイド化（デザインシステム化）。
- 必要なら個別SVGとして書き出し（`/icon/svg` など）。
